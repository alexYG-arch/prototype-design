# Information Presentation Consistency

## General rule

在相同语义意图、触发条件、作用范围和信息生命周期下，应采用一致且唯一的信息呈现方式。差异化呈现必须由明确的场景差异、用户决策需求、严重程度、持续性、恢复义务或承载面约束支持，并记录依据。

该规则适用于但不限于：页面内容区、结果区、空状态、局部提示、横幅、短暂反馈、对话框、浮层、气泡、标记、系统通知和控件状态。

## Presentation intent

每个信息意图应描述：

```text
触发事件
影响范围
严重程度
是否阻断
是否要求决策
持续时间
恢复操作
首选表现模式
```

## Consistency checks

- 同一意图是否具有多个并行主表现；
- 同一恢复操作是否无依据重复；
- 需要持续处理的信息是否只使用短暂表现；
- 非阻断信息是否被不必要地升级为阻断表现；
- 相同语义跨页面是否随机漂移；
- PRD 已明确表现方式时，是否先校验后采纳；
- 差异化表现是否记录理由。
