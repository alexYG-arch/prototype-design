# Authority Model

## Authority order

```text
Source PRD / approved platform constraints
→ Human decisions
→ Locked DRD Constitution and Contracts
→ Approved upstream Stage artifacts
→ Rules and Projection Rules
→ Skills and Workpacks
→ Candidate implementation or document output
```

## Ownership

- PRD：产品目标、范围、产品能力、业务规则、资格与验收标准。
- DRD：用户任务、页面、承载面、交互状态、元素、表现模式与布局说明。
- Python Control Plane：Snapshot、DAG、Hash、Validation、Review、Promotion、Invalidation、Compilation。
- Codex Runtime：语义分类、演绎推理、候选自然语言、代码 Candidate。
- Human：范围争议、归纳候选、关键页面与交互、最终批准。

## Cross-boundary behavior

DRD 发现产品缺口时输出：

```text
DRD_TO_PRD_GAP
DRD_TO_PRD_QUESTION
```

不得直接改 PRD。
