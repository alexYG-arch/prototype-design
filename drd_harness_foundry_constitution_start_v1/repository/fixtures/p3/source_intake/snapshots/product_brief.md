# Operations Console Source Brief

## Purpose

Create a compact operational console for reviewing submitted cases, their intake state, and required reviewer actions.

## Source Boundaries

The brief defines the source material for P3 source intake validation only. It is not an approval to add new product capabilities during intake.

## Required Source Facts

- Cases can be submitted for review.
- Each case has an intake state.
- Reviewers need to see whether a case is ready, blocked, or waiting for human review.
- Missing product details must remain review gaps until explicitly approved.
